# 原项目关系模型考察与本项目适配

日期：2026-07-13

## 1. 原项目的根本单位

根据作者公开方法说明和运行界面，主视图不是“人物关系图”，而是“句子/观点时间轴”：

- 人物是标题、肖像和年代组成的观点组容器。
- 一个人物可以有多条按论证顺序排列的观点；不能用一个圆点替代其整个体系。
- 每条观点是独立节点，可有主题、关键词、出处和多条关系。
- 绿色关系表示同意、相似或发展；红色关系表示分歧、对照或反驳。
- 主视图关系通常是无向的概念关系，不自动表示师承、阅读、传播或历史影响。
- 人物图和人物时间轴属于由观点关系聚合出的二级视图，不应反过来成为权威数据。

人物聚焦显示该人物的全部观点及其一度关联观点；观点聚焦显示该观点及其一度邻居。总览中的标题和观点文字并未删除，而是随画布缩小为短线与色块；放大后重新成为可读文字、标签和出处。入门筛选是人工挑选每位人物少量核心观点，不是按知名度自动生成。

## 1.1 原站布局实测

2026-07-14 使用浏览器对原站运行画布做只读测量，得到以下可复现结论：

- 主图由两个全屏 Canvas 绘制，不存在可见 SVG 时间轴。
- 人物索引共 189 项；人物标题按出生年份排序，但相邻人物距离不按年份差成比例。
- 人物标题全部落在同一条斜向内容坐标上。实测基础标题槽约为 `53.33` 世界单位，每条观点再增加约 `20.06` 世界单位，拟合误差约 `0.39` 世界单位。
- 因而下一人物的位置由“当前人物标题槽 + 当前人物观点行数”决定：`next = current + personSlot + statementCount × statementSlot`。
- 这解释了原站为何既有年代顺序又不因历史空档变得稀疏：年份负责排序与显示，观点数量负责内容占位。

本项目使用严格45°的左上至右下方向：人物基础步长为 `54 × (1, 1)`，每条观点步长为 `40 × (1, 1)`。界面不绘制实体轴线或年代刻度。总览让人物标题和观点文字随画布共同压缩成点列与短线纹理；聚焦后只保留严格一度关系，并按可见人物与观点重新紧凑排版。

## 2. 本项目的适配模型

```text
Person（人物/体系容器）
  └─ Statement 1（观点节点）──── positive / negative ──── Statement X
  └─ Statement 2（观点节点）──── positive / negative ──── Statement Y
  └─ Statement 3（观点节点）
```

公开边只能来自逐条审核的 `statement_relations.csv`。同传统、同领域、同标签只能产生后台候选。`polarity` 描述概念关系，`historical_influence` 单独描述历史影响，二者不得混用。

截至本次考察，没有发现原项目发布的官方源码仓库或允许复制代码、数据、肖像和视觉素材的开源许可。本项目只参考作者公开描述的知识组织和交互原则，使用独立代码、中国哲学数据、分类体系和视觉实现。
