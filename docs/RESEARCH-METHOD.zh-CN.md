# 一期研究方法与纳入标准

版本：`phase1-2026-07-13`  
适用范围：人物基础名录、观点、观点关系、来源与双语研究留痕

## 1. 项目研究单位

本项目以“可定位、可引用的思想观点”为知识节点，以人物作为观点体系和时间容器。第一期已经建立人物基础名录、显示系统，以及一组小规模、逐条审核的观点—关系种子；任何未经审核的关系都不得公开绘制。

原型参照项目 *History of Philosophy* 的公开方法不是“收集名言”，而是从原典或可靠二手文献中提炼能够独立表达观点、论证或立场的句子，记录来源、分类与关系。本项目沿用这一研究尺度，但使用独立数据、分类和实现。

## 2. 纳入条件

候选人至少满足以下一项，并且必须能找到可靠研究入口：

1. 其著作、言论或教学传统形成了中国哲学史反复讨论的观点或问题。
2. 其政治、宗教、文学、史学、科学或翻译实践对中国哲学议题产生了可论证的影响。
3. 虽非可考历史人物，但作为经典文本的稳定作者人格或思想人格长期参与哲学讨论。
4. 当代人物提出具有辨识度的哲学体系、论证或解释框架，而非仅从事一般性学术工作。

一期分三种纳入层级：

- `core`：标准哲学史或专题研究持续讨论的核心人物。
- `extended`：在相邻领域形成可定位哲学问题的扩展人物。
- `context`：经典叙事、文本归属或思想谱系不可绕开的语境人物。

## 3. 明确排除

- 只有一句网络名言或励志语、没有稳定文本与学术讨论的人物。
- 仅因身份显赫、政治职位、知名度或道德评价而入选的人物。
- 仅重复既有观点、尚无独立贡献或解释影响的一般研究者。
- 无法消除同名歧义、无法给出年代依据或找不到可靠研究入口的条目。

“不纳入”不是对人物价值的评价，只说明其尚未满足本图谱的可记录哲学标准。

## 4. 历史真实性分层

| 值 | 含义 |
|---|---|
| `attested` | 有可考生平或活动记录的历史人物 |
| `traditional` | 传统归属较稳定，但生平或部分归属存在争议 |
| `textual-persona` | 主要作为文本作者人格出现，作者或成书问题复杂 |
| `legendary` | 神话—传说人格，不能按普通历史人物理解 |

伏羲、黄帝等可以作为思想史中的传统人格进入图谱，但页面必须显示其类型与传统纪年，禁止把传统纪年写成考古事实。

## 5. 审核状态

- `screened`：已通过纳入标准初筛，具有研究入口；年代、归属和内容仍可继续精校。
- `source-reviewed`：编辑已依据列出的学术来源复核本期纳入和基本分类；不表示人物的所有命题均已审核。

每个观点和每条关系都有独立审核状态，不能继承人物记录的审核状态。观点必须记录人物、顺序、原典位置、双语摘要和解释；关系必须记录两端观点、正负性质、子类型、判断依据、历史影响状态、证据来源、编辑者和时间。

研究进度单独保存在 `data/reports/research-queue.csv`。队列行不是观点，绝不能显示成页面节点或成为关系端点。

## 6. 一期覆盖

一期人物基线包含 155 人：传说上古 8、先秦 25、秦汉 10、魏晋隋唐 30、宋元 16、明清 22、晚清民国 20、二十世纪 13、当代 11。其中 138 人标记为可考人物，12 人属于传统或神话人格，5 人属于文本人格；71 条为来源已复核，84 条为初步筛选。

Schema v4 当前包含155人名下167条逐条带来源定位的双语观点；每人至少1条，原有6位核心人物保留多观点结构。14条已复核观点关系中正向10条、负向4条；一期关系仍是种子集，二期再扩充有证据的观点—观点关系。

“155 人”是公开版本的可审计基线，不是“已经穷尽中国哲学”的声明。新增、删除和改级都应形成版本记录。

## 7. 观点关系规则

- 人物不是边的端点；关系的两端必须是两个不同的观点 ID。
- `positive` 表示相似或扩展，`negative` 表示对照或反驳；颜色不表示时间方向。
- 默认关系为 `undirected`，说明概念可比较，不自动声称后人读过前人或受到影响。
- `historical_influence` 与概念关系分开记录；只有证据充分时才能标为可能或明确影响。
- `basis` 区分原文明确回应、思想史研究判断与编辑性概念比较。
- 同传统、同领域、同标签等自动共现只能生成后台候选，不能直接写入公开关系表。
- 人物聚焦显示该人物全部观点及其一度邻居；观点聚焦显示该观点及其一度邻居。

## 8. 数据维护

维护人员只编辑：

- `data/people.csv`：非语言事实、分类、审核状态、来源指向。
- `data/people_i18n.csv`：中英文姓名和别名。
- `data/statements.csv`：观点事实、人物归属、顺序、领域与审核状态。
- `data/statement_translations.csv`：每个 locale 一行的观点文本、解释和标签。
- `data/statement_sources.csv`：观点的来源 ID、精确篇章位置和证据角色。
- `data/statement_relations.csv`：观点两端、正负性质、子类型、依据、影响状态和证据来源。
- `data/relation_translations.csv`：每个 locale 一行的关系说明。
- `data/taxonomies.csv`：时期、传统和领域的双语词表。
- `data/sources.csv`：研究入口和引用信息。

执行 `npm run data:build` 后，系统会校验 ID、双语字段、年代、来源和分类，再自动生成前端数据；坐标和标签布局由年份和碰撞算法计算，无需手绘网页。
